## 1. 这个滤波器是干什么的

互补滤波器用于**融合两个传感器**的数据,各取所长:

| 传感器 | 优点 | 缺点 | 擅长频率 |
|--------|------|------|-----------|
| 加速度计 | 长期稳定,不漂移 | 短期噪声大,受振动影响 | 低频(准) |
| 陀螺仪 | 短期精度高,响应快 | 长期积分漂移 | 高频(准) |

互补滤波的核心思想: **加速度计信任低频部分,陀螺仪信任高频部分**,两者互补后得到一个既无噪声又无漂移的角度估计。

**典型应用**: 平衡车、手持云台、四轴飞行器姿态估计、航海姿态参考系统(AHRS)。

## 2. 为什么用互补滤波而不是卡尔曼

这是嵌入式MCU上一个永恒的选择题。以下是对比:

| 维度 | 互补滤波 | 卡尔曼滤波 |
|------|---------|-----------|
| **计算量** | 1次乘法 + 1次加法 | 矩阵运算(至少几十次乘加) |
| **RAM需求** | 1个结构体 ~12字节 | 协方差矩阵P(2x2)等 ~几十字节 |
| **代码量** | 十几行 | 上百行 |
| **调参难度** | 1个参数α | R/Q矩阵,需要理解噪声统计 |
| **最优性** | 经验近似最优 | 理论最优(线性高斯) |
| **适用场景** | 资源受限MCU | 算力充足的MPU/PC |

**结论**: 在MSPM0G3507这种主频32MHz、RAM几十KB的MCU上,互补滤波是唯一现实的选择。卡尔曼的矩阵运算在这里会吃掉大量CPU时间,而且调参不当效果反而不如互补滤波。

> **显然的事实**: 互补滤波本质上是**固定增益的简化卡尔曼滤波器**。当卡尔曼增益收敛到稳态后,两者数学形式等价。互补滤波相当于跳过了卡尔曼的增益迭代过程,直接使用经验增益。

## 3. 结构体定义逐行拆解

```c
typedef struct {
    float angle;    //融合后的角度估计值
    float bias;     //陀螺仪零偏估计
    float alpha;    //互补系数,alpha越大越信任陀螺仪
} Complementary_t;
```

**逐行解释**:

| 成员 | 类型 | 含义 | 初始值 | 取值范围 |
|------|------|------|--------|----------|
| `angle` | float | 当前融合后的角度估计值(度或弧度,由外部统一) | 0.0f | (-∞, +∞) |
| `bias` | float | 陀螺仪的零偏(静态误差),单位与角速度相同 | 0.0f | 通常很小,~0.01量级 |
| `alpha` | float | 互补系数,权重分配 | 由Init传入 | [0, 1] |

**关于alpha的命名困惑**:

注释说"alpha越大越信任陀螺仪",但在Update函数中实际的公式是:

```c
cf->angle = cf->alpha * angle_acc + (1.0f - cf->alpha) * gyro_integ;
```

所以:

- **alpha越接近1** → `angle_acc`权重越大 → **越信任加速度计** → 输出跟随加速度计,响应快但噪声大
- **alpha越接近0** → `gyro_integ`权重越大 → **越信任陀螺仪** → 输出平滑但可能出现漂移

**显然的问题**: 注释和实现中alpha的语义是相反的。注释写"alpha越大越信任陀螺仪",但代码里alpha乘的是加速度计项。这是一个**文档与实现不一致**的典型案例。实际使用时以代码为准: **alpha乘的是加速度计角度**。

> **如果你想让alpha的语义变成"越大越信任陀螺仪"**,只需交换加权系数:
> ```c
> cf->angle = (1.0f - cf->alpha) * angle_acc + cf->alpha * gyro_integ;
> ```
> 这是一行代码的事,但一旦修改,所有使用该模块的地方都要同步理解。

## 4. Complementary_Init 逐行拆解

```c
void Complementary_Init(Complementary_t *cf, float alpha)
{
    if (cf == NULL) return;
    if (alpha < 0.0f) alpha = 0.0f;
    if (alpha > 1.0f) alpha = 1.0f;
    cf->alpha = alpha;
    cf->angle = 0.0f;
    cf->bias  = 0.0f;
}
```

**逐行解释**:

- `if (cf == NULL) return;` — 防御性编程。如果传入了空指针,直接返回不做任何操作。否则对NULL解引用会触发HardFault。

- `if (alpha < 0.0f) alpha = 0.0f;` — 钳位下限。alpha不能为负,负值没有物理意义。

- `if (alpha > 1.0f) alpha = 1.0f;` — 钳位上限。alpha超过1会导致输出大于输入,滤波器不再稳定。

- `cf->alpha = alpha;` — 写入钳位后的alpha。注意这里**直接赋值,没有做 `1.0f - alpha` 的预计算**,意味着每次Update都要重新计算 `(1.0f - cf->alpha)`,浪费了一个浮点运算。优化方案是在Init里预先算好 `one_minus_alpha` 存起来。

- `cf->angle = 0.0f;` — 角度初始化为0。如果系统启动时并非水平,这里会有**启动阶跃误差**,需要调用Complementary_Reset重新设置初始角度。

- `cf->bias = 0.0f;` — 偏置初始化为0。这隐含假设陀螺仪在静止时输出恰好为0。实际芯片标称零偏可能达到 ±5°/s,必须在使用前进行校准。

## 5. Complementary_SetAlpha 逐行拆解

```c
void Complementary_SetAlpha(Complementary_t *cf, float alpha)
{
    if (cf == NULL) return;
    if (alpha < 0.0f) alpha = 0.0f;
    if (alpha > 1.0f) alpha = 1.0f;
    cf->alpha = alpha;
}
```

**逐行解释**:

与Init中的alpha处理完全相同。这个函数存在的意义是**运行时动态调整融合权重**。例如:

- 系统静止时 → 增大alpha → 多信任加速度计 → 快速消除陀螺漂移误差
- 系统剧烈运动时 → 减小alpha → 多信任陀螺仪 → 避免加速度计受到运动加速度干扰

这种动态调整称为**自适应互补滤波**,但在这个模块中需要外部手动调用SetAlpha实现,模块本身没有做自适应逻辑。

## 6. Complementary_Reset 逐行拆解

```c
void Complementary_Reset(Complementary_t *cf, float init_angle)
{
    if (cf == NULL) return;
    cf->angle = init_angle;
    cf->bias  = 0.0f;
}
```

**逐行解释**:

- `cf->angle = init_angle;` — 把角度硬设置为指定值。一般用在系统检测到静止时,直接用加速度计计算的角度覆盖当前估计,消除长期运行后积累的误差。

- `cf->bias = 0.0f;` — 同时清零偏置。这意味着重置后陀螺积分重新从零偏开始。

**显然的缺失**: Reset没有重置alpha。如果你之前改过alpha,Reset后alpha保持不变。这不是bug,这是设计选择——alpha是"配置",angle和bias是"状态"。

## 7. Complementary_Update 逐行拆解

```c
float Complementary_Update(Complementary_t *cf, float angle_acc,
                           float rate_gyro, float dt)
{
    if (cf == NULL) return 0.0f;
    if (dt <= 0.0f) return cf->angle;
```

**入口参数**:

| 参数 | 含义 | 来源 | 单位 |
|------|------|------|------|
| `angle_acc` | 当前加速度计计算的倾角 | atan2(acc_y, acc_z) 等 | 度或弧度 |
| `rate_gyro` | 当前陀螺仪角速度 | 陀螺仪原始值 × 灵敏度 | °/s 或 rad/s |
| `dt` | 上次Update到这次的时间间隔 | 定时器测量 | 秒 |

**防御逻辑**:

- `if (cf == NULL) return 0.0f;` — 空指针直接返回0。注意这里不像其他函数返回`cf->angle`(无法访问),而是返回硬编码0。

- `if (dt <= 0.0f) return cf->angle;` — dt必须为正。如果dt=0(两次Update在同一时刻调用),积分步长为零,陀螺积分不变,直接返回当前角度。这个检查**防止了除以零或积分异常**。

```c
    float gyro_integ = cf->angle + (rate_gyro - cf->bias) * dt;
```

**这是陀螺仪积分步骤**:

- `rate_gyro - cf->bias` — **减去零偏校正**。陀螺仪存在静态漂移,减去bias后得到校正后的角速度。如果bias=0(初始化默认值),这里就是原始角速度。

- `(rate_gyro - cf->bias) * dt` — 角速度 × 时间 = 角度增量。这是最简单的**欧拉积分**,一阶精度。

- `cf->angle + ...` — 累加到当前角度上。这是**开环积分**,误差会随着时间累积。

**欧拉积分的局限性**: 当角速度快速变化时,欧拉积分精度不如龙格-库塔(RK4),但在MCU上欧拉积分计算量最小。只要采样频率足够高(>>运动带宽),误差可接受。

```c
    cf->angle = cf->alpha * angle_acc + (1.0f - cf->alpha) * gyro_integ;
```

**这是融合步骤**:

- `cf->alpha * angle_acc` — 加速度计角度的加权分量
- `(1.0f - cf->alpha) * gyro_integ` — 陀螺积分角度的加权分量

**两者的和就是互补滤波输出的最终角度**。

**为什么这样能互补?**

从频域角度看:
- 加速度计贡献经过**低通** — alpha < 1,所以加速度计的高频噪声被衰减
- 陀螺仪贡献经过**高通** — 1-alpha 加权,陀螺仪的低频漂移被衰减(因为长时间积分误差占主导,但其短时响应被保留)

两者叠加,整体呈现**全通**特性——低频靠加速度计,高频靠陀螺仪。

```c
    return cf->angle;
}
```

返回融合后的角度估计值。

## 8. 数学公式全解析

### 8.1 离散时间递推公式

完整的互补滤波离散递推方程为:

```
θ[n] = α · θ_acc[n] + (1-α) · [θ[n-1] + (ω[n] - b) · Δt]
```

其中:

| 符号 | 含义 |
|------|------|
| θ[n] | 当前时刻融合角度输出 |
| θ[n-1] | 上一时刻融合角度 |
| θ_acc[n] | 当前加速度计计算角度 |
| ω[n] | 当前陀螺仪角速度 |
| b | 陀螺零偏估计 |
| Δt | 采样间隔 |
| α | 互补系数 |

### 8.2 连续时间传递函数

对上述离散方程做Z变换并近似连续化,得到**一阶高通+一阶低通**的并联结构:

```
            α · s + (1-α)/τ
H(s) = ──────────────────────
               s + (1-α)/τ
```

其中 τ = Δt。转折频率:

```
f_c = (1-α) / (2π · Δt)
```

这个转折频率是**加速度计和陀螺仪信任度相等的频率点**:
- 低于 f_c → 更信任加速度计
- 高于 f_c → 更信任陀螺仪

### 8.3 转折频率与α的关系

| α | 转折频率 f_c | 特性 |
|---|-------------|------|
| 0.98 | f_c = 0.02/(2π·Δt) | 极低,几乎完全信任陀螺仪 |
| 0.90 | f_c = 0.10/(2π·Δt) | 适度信任陀螺仪 |
| 0.50 | f_c = 0.50/(2π·Δt) | 两者等权,转折点居中 |
| 0.10 | f_c = 0.90/(2π·Δt) | 非常信任加速度计 |

**举例**: 采样率100Hz(Δt=0.01s), α=0.98:

```
f_c = 0.02 / (2π × 0.01) ≈ 0.32 Hz
```

这意味着频率低于0.32Hz的信号主要来自加速度计,高于0.32Hz的来自陀螺仪。

## 9. 参数选择指导

### 9.1 α的选择

α是互补滤波器中**唯一需要调的参数**,它是整个滤波器行为的灵魂。

**调整原则**:

```
α越大 → 输出越像加速度计 → 无漂移但有噪声 → 适合静止/慢速运动
α越小 → 输出越像陀螺仪 → 平滑但可能漂移 → 适合快速动态运动
```

**典型值参考**:

| 应用场景 | 推荐α | 原因 |
|---------|-------|------|
| 平衡车 | 0.95 ~ 0.98 | 车体倾斜变化较慢,需要抑制加速度计噪声 |
| 手持云台 | 0.85 ~ 0.95 | 快速转动时需要陀螺仪占主导 |
| 飞行器姿态 | 0.90 ~ 0.96 | 悬停时多信任加速度计,机动时信任陀螺仪 |
| 航海姿态 | 0.98 ~ 0.995 | 船体运动极慢,几乎完全依赖陀螺仪 |

**调试步骤**:

1. 从α=0.50开始,让加速度计和陀螺仪各占一半
2. 用手快速晃动传感器,观察输出的平滑度
3. 静止放置传感器,观察输出是否漂移
4. 逐渐增大α,找到"快速响应"和"不漂移"之间的平衡点

### 9.2 采样间隔dt的影响

**显然的事实**: dt的精度直接影响互补滤波的性能。

- dt太大(>0.1s) → 陀螺仪积分步长过大 → 欧拉积分误差爆炸
- dt太小(<0.001s) → 陀螺每步积分角度太小 → 浮点精度损失, (1-α) 非常接近1

**推荐**: dt在 0.001s ~ 0.02s 之间。

### 9.3 陀螺零偏bias的校准

**校准方法**:

1. 传感器静止放置
2. 采集N秒的陀螺仪输出,求平均值
3. 这个平均值就是零偏bias

在代码中,可以在Init后调用:

```c
Complementary_Reset(&cf, 0.0f);
cf.bias = estimated_bias;  // 直接写结构体
```

**为什么需要校准**: 典型的MEMS陀螺仪(如MPU6050)出厂零偏可达 ±5°/s。不校准的话,一分钟后积分角度误差可达300°。

## 10. 典型调用链

```
main()
├── Board_Init()                 // 板级初始化
├── I2C_Init()                   // 初始化I2C总线连接IMU
├── IMU_Init()                   // 初始化加速度计/陀螺仪
├── Complementary_Init(&cf, 0.98) // 初始化互补滤波器
│
└── while(1)
    ├── dt = GetTimerDelta()     // 计算时间差
    ├── accel = ReadAccel()      // 读取加速度计原始值
    ├── gyro = ReadGyro()        // 读取陀螺仪原始值
    ├── angle_acc = atan2(accel.y, accel.z) * RAD2DEG  // 加速度计计算倾角
    ├── angle_fused = Complementary_Update(&cf, angle_acc, gyro, dt)  // 互补滤波
    ├── SetMotorSpeed(PID_Update(angle_fused, target_angle))  // 送PID控制器
    └── delay(10ms)
```

**被谁调用**:
- **直接调用者**: 姿态解算任务(`attitude_estimate()`)、传感器融合任务
- **间接使用者**: PID控制器(接收融合后的角度作为反馈)、上层逻辑控制

**调用频率**: 通常与IMU读取频率一致,典型值 50Hz ~ 200Hz。

注意注释中写的是"调用者:",后面留空了——这是因为代码作者把这个信息留给开发者按实际情况填写。

## 11. 踩坑点大全

### 11.1 坑1: 长期积分后的陀螺零漂

**现象**: 运行几分钟后,静止时的角度输出缓慢漂移,不再回到0。

**根因**: `bias`没有更新。陀螺仪存在温漂,刚开机时校准的bias,运行几分钟后芯片发热,bias发生变化。

**解决方案**:

(1) **静止检测 + 自动零偏校准**:

```c
if (is_stationary(gyro, accel)) {
    // 静止时用加速度计角度强制修正
    cf->angle = angle_acc;
    // 同时更新bias
    cf->bias = 0.9f * cf->bias + 0.1f * gyro;  // 一阶LPF更新bias
}
```

(2) **互补滤波本身带高通特性**: 如果α足够大(>0.98),陀螺仪高通路径会自然抑制低频漂移,但不能完全消除。

### 11.2 坑2: 角度单位不统一

**现象**: 融合后角度一会儿在±180范围,一会儿突然跳到359。

**根因**: `angle_acc` 用 `atan2` 返回范围是 `-π~π` 或 `-180°~180°`,但陀螺积分是连续累积,可能超过这个范围。两者直接相加/加权时出现**角度环绕**问题。

**解决方案**: 在Update之前确保两个角度在同一个连续范围内,或者使用四元数避免欧拉角的万向锁和环绕问题。

### 11.3 坑3: 运动加速度干扰

**现象**: 快速加减速时,融合角度出现毛刺。

**根因**: 加速度计测量的加速度包含**重力分量+运动加速度分量**。当系统有水平加速度时,`atan2`计算的角度被污染。

**解决方案**: 使用陀螺仪在运动期间占主导(减小alpha),静态时增大alpha。这需要外部逻辑判断运动状态。

### 11.4 坑4: α太接近0的稳定性

**现象**: α=0.001时,输出几乎完全来自陀螺积分,几分钟后漂移到不可用。

**根因**: 陀螺仪开环积分没有任何校正机制。

**解决方案**: α不要小于0.50。如果确实需要"几乎全部信任陀螺仪",应该考虑加额外的零偏校正机制。

### 11.5 坑5: dt的抖动

**现象**: 使用`delay`或操作系统调度导致的dt不规则,角度输出出现步进跳变。

**根因**: `dt`进入陀螺积分公式 `(rate_gyro - cf->bias) * dt`,dt抖动直接转换为角度抖动。

**解决方案**:

1. 使用硬件定时器精确测量两次调用之间的时间
2. 对dt做一阶低通滤波去除抖动
3. 固定采样率,使用恒定的dt(如系统滴答定时器中断触发)

## 12. 初始化与启动瞬态

### 12.1 初始值影响

| 结构体成员 | 初始值 | 启动后的表现 |
|-----------|--------|-------------|
| `angle` | 0.0f | 如果实际角度≠0,输出需要时间从0过渡到真实值 |
| `bias` | 0.0f | 如果实际bias≠0,陀螺仪积分有恒定偏移 |
| `alpha` | 用户设定 | 固定,除非运行中修改 |

### 12.2 启动瞬态

**问题**: 从 `angle=0` 开始,需要多少时间输出稳定?

**定量分析**:

互补滤波的时间常数:

```
τ = Δt / (1-α)
```

举例: Δt=0.01s(100Hz), α=0.98:

```
τ = 0.01 / 0.02 = 0.5s
```

这意味着**启动后约 3τ = 1.5s 输出进入稳态**。

如果α=0.50:

```
τ = 0.01 / 0.50 = 0.02s
```

**不到0.1s就稳定了**。

所以α越大,启动过渡越长。对于实时系统,可以在启动后先用较小α,稳定后再切换到目标α:

```c
Complementary_Init(&cf, 0.50);  // 先用0.5快速收敛
for (int i = 0; i < 100; i++) {
    // 运行100次滤波
}
Complementary_SetAlpha(&cf, 0.98);  // 切换到目标值
```

### 12.3 正确的初始化流程

```c
Complementary_t cf;

// 1. 初始化结构体
Complementary_Init(&cf, 0.98f);

// 2. 如果是静止启动,读取加速度计初始化角度
float init_angle = atan2(acc_y, acc_z) * RAD2DEG;
Complementary_Reset(&cf, init_angle);

// 3. (可选)写入预校准的陀螺零偏
cf.bias = CALIBRATED_BIAS;

// 4. 开始循环
while (1) {
    // ...
}
```

## 13. 调试手段

### 13.1 串口打印对比

这是最基本的调试方法。同时打印原始加速度计角度和融合角度:

```c
printf("%lu,%f,%f,%f\n", micros(), angle_acc, angle_fused, gyro);
```

用串口助手或Python脚本接收,绘制两条曲线对比。

**观察要点**:

- 静止时: 两条线都应该稳定在0附近,融合后曲线更平滑
- 快速转动时: 融合曲线应该跟随陀螺仪响应,没有滞后
- 长时间后: 融合曲线不应像陀螺积分那样漂移

### 13.2 多参数扫描

如果想找到最优α,让上位机通过串口发送命令修改:

```c
// 串口接收回调
void UART_ReceiveCmd(char cmd, float param)
{
    if (cmd == 'a') {
        Complementary_SetAlpha(&cf, param);
        printf("alpha set to %f\n", param);
    }
}
```

然后在上位机写脚本,自动扫描α从0.90到0.99,每个值运行10秒,记录RMS误差,找到最优值。

### 13.3 中间变量输出

除了最终角度,还可以输出中间变量帮助理解:

```c
// 调试版Update
float gyro_integ = cf->angle + (rate_gyro - cf->bias) * dt;
cf->angle = cf->alpha * angle_acc + (1.0f - cf->alpha) * gyro_integ;

// 打印中间变量
printf("%f,%f,%f,%f\n", angle_acc, gyro_integ, cf->angle, dt);
return cf->angle;
```

## 14. 为什么都用float

**显然的问题**: 为什么不用double? 为什么不用定点数?

**原因分析**:

| 类型 | 精度 | 占空间 | MSPM0G3507硬件支持 | 推荐? |
|------|------|--------|-------------------|-------|
| `float` | 6-7位有效数字 | 4字节 | 有FPU,单精度硬件 | ✅ 推荐 |
| `double` | 15-16位有效数字 | 8字节 | 无双精度FPU(软仿) | ❌ 太慢 |
| `int32_t` 定点 | 自定义 | 4字节 | 硬件整数 | ❌ 开发麻烦 |

**关键事实**: MSPM0G3507有**单精度浮点单元(FPU)**,执行float运算与整数运算速度相当。使用float:

- 代码简洁,可读性好
- 硬件加速,性能足够
- 4字节足够支撑传感器融合精度(加速度计/陀螺仪精度本身只有16位ADC,float的24位尾数绰绰有余)

**为什么不显式标注`#include <math.h>`?** 该模块没有使用sin/cos/sqrt等数学函数,只需要最基本的加减乘除,这些由编译器内置处理,不需要额外包含头文件。
