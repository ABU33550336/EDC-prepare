# line_follower 模块 — 循迹偏差计算（加权重心法）

> **当你看代码旁边的注释都看不懂时来翻阅的百科全书**
> 适用芯片：MSPM0G3507 @ 32MHz | 纯 C | 零硬件依赖

---

## 1. 模块概述

这个模块不做 PID 控制，不做电机驱动，只做一件事：**根据灰度传感器的原始读数，算出一条线相对于小车中心线的偏离量**。

简单说：8 个灰度传感器排成一排，扫到黑色的线就输出高电平/大阻值。哪个传感器离黑线最近？偏差是偏左还是偏右？偏了多少？——这就是这个模块唯一要回答的问题。

### 使用场景
- 黑线循迹小车：白底黑线或黑底白线
- 输入：8 路灰度传感器原始 ADC 值
- 输出：归一化偏差 (-100 ~ +100)，可直接喂给 PID

---

## 2. 常量定义

```c
#define LINE_SENSOR_COUNT 4
```

### 为什么是 4 而不是 8？

根据项目描述，"8路灰度传感器"是硬件配置，但代码里只定义了 4 路。有两种可能：

1. **代码与硬件不完全匹配**：目前只实现了 4 路的算法，后续需要扩展到 8 路
2. **硬件实际只有 4 路**：项目需求文档说的是 8 路但物理上只装了 4 个

**建议**：如果硬件是 8 路传感器，修改 `LINE_SENSOR_COUNT` 为 8，同时扩展 `LINE_POSITIONS` 数组为 `{-7, -5, -3, -1, 1, 3, 5, 7}`。

### 传感器位置权重

```c
static const int16_t LINE_POSITIONS[LINE_SENSOR_COUNT] = {
    -3, -1, 1, 3
};
```

这是 **4 路传感器**从左到右的位置权重。物理含义：

```
传感器索引:    0       1       2       3
位置权重:    -3      -1      +1      +3
              |       |       |       |
           左边界  左中     右中   右边界
```

权重间隔为 2（相邻传感器间距的 2 倍），中心点在 0 位置。如果线正好在传感器 1 和 2 之间（即车体中心正下方），偏差为 0。

## 3. 数学原理：加权重心法（Weighted Centroid）

### 3.1 公式

```
偏差 = [ Σ (position_i × weight_i) / Σ weight_i ] / 3 × 100
```

| 符号 | 含义 |
|------|------|
| position_i | 第 i 个传感器的位置权重 |
| weight_i | 第 i 个传感器的权重（原始 ADC 值或二值化标志） |
| 分母 | 总权重（所有传感器权重之和） |
| 归一化系数 | 除以 3（最大位置值）再乘 100 |

### 3.2 直观理解

这本质上是求一个**加权平均位置**：每个传感器的位置权重乘以该传感器的读数，然后取平均。结果就是"黑线的等效位置"。

类比：
- 桌子上有 4 个支点，位置分别是 -3, -1, 1, 3
- 每个支点上压了 weight_i 克的重物
- 重心位置在哪？——加权平均位置

### 3.3 为什么除以 3？

- 位置权重范围：[-3, +3]
- 加权平均位置范围：[-3, +3]
- 归一化到 [-100, +100] 需要除以 3 再乘以 100

**所以偏差范围是 [-100, +100]，表示偏离中心线的百分比**：
- 0：正对黑线
- +100：线在最右侧（极端偏右）
- -100：线在最左侧（极端偏左）

### 3.4 传感器数量扩展后的归一化系数

如果扩展到 8 路传感器，位置数组为 `{-7, -5, -3, -1, 1, 3, 5, 7}`：
- 最大位置值为 7
- 归一化系数变为 1/7 × 100
- 需要把第 47 行的 `3.0f` 改为 `7.0f`

## 4. 两种工作模式

### 4.1 Mode 0：模拟量加权

```c
weight = (float)sensor_values[i];
```

把 ADC 原始读数直接作为权重。

**优点**：保留了灰度值的信息量。线边沿的传感器读数处于"半黑半白"状态时，可以做到亚传感器精度的偏差检测。

**缺点**：受环境光影响大。同样一条黑线，在不同光照下 ADC 值可能差 3 倍。

**典型 ADC 值**：
- 白底：~3000（假设 12-bit ADC, 0-4095）
- 黑线：~500
- 纯黑不反光：~100

### 4.2 Mode 1：二值化加权

```c
weight = (sensor_values[i] >= threshold) ? 1.0f : 0.0f;
```

超过阈值视为"检测到线"（权重=1），否则视为"没检测到"（权重=0）。

**优点**：抗光照干扰。只要阈值设置合适，白天黑夜都能工作。

**缺点**：丢失了灰度信息。线正好在两个传感器之间时，只能检测到"其中之一"而不是"中间位置"。

### 4.3 什么时候用哪种模式？

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 环境光稳定（室内竞赛） | Mode 0 | 精度更高，可实现亚传感器分辨率 |
| 环境光变化大（室外/强弱光交替） | Mode 1 | 阈值固定，抗干扰能力强 |
| 黑线宽度 > 传感器间距 2 倍 | Mode 0 或 1 | 都差不多 |
| 黑线宽度 < 传感器间距 | Mode 0 | 二值化可能在两传感器间漏检测 |

## 5. 阈值标定

```c
// 二值化阈值标定方法
#define THRESHOLD_BLACK_WHITE  1500    // 示例值

// 标定步骤:
// 1. 把传感器放在白底上, 读值: 比如 white_val = 2800
// 2. 把传感器放在黑线上, 读值: 比如 black_val = 400
// 3. 阈值取中间值: threshold = (white_val + black_val) / 2 = 1600
```

**注意**：阈值随光照和传感器型号变化。**每次更换场地需要重新标定**。

### 自动阈值标定算法（建议实现）

```c
uint16_t auto_calibrate_threshold(void)
{
    uint16_t max_val = 0, min_val = 0xFFFF;
    // 转动小车扫过黑线和白底区域
    for (int i = 0; i < 100; i++) {
        uint16_t val = read_sensor(0);
        if (val > max_val) max_val = val;
        if (val < min_val) min_val = val;
    }
    return (max_val + min_val) / 2;
}
```

## 6. 函数详解

### 6.1 `LineFollower_GetDeviation`

```c
float LineFollower_GetDeviation(const uint16_t *sensor_values,
                                uint16_t threshold,
                                uint8_t mode)
{
    if (sensor_values == NULL) return 0.0f;

    float weighted_sum = 0.0f;
    float total_weight = 0.0f;

    for (uint8_t i = 0; i < LINE_SENSOR_COUNT; i++) {
        float weight = 0.0f;

        if (mode == 0) {
            weight = (float)sensor_values[i];
        } else {
            weight = (sensor_values[i] >= threshold) ? 1.0f : 0.0f;
        }

        weighted_sum += (float)LINE_POSITIONS[i] * weight;
        total_weight += weight;
    }

    if (total_weight < 0.001f) {
        return 0.0f;
    }

    float deviation = (weighted_sum / total_weight) / 3.0f * 100.0f;

    if (deviation > 100.0f) deviation = 100.0f;
    if (deviation < -100.0f) deviation = -100.0f;

    return deviation;
}
```

**逐行详解：**

| 行号 | 代码 | 解释 |
|------|------|------|
| 23 | 空指针检查 | sensor_values 为 NULL 时返回 0（表示无偏差），不崩溃 |
| 25-26 | 累加器初始化 | weighted_sum 存放 Σ(p_i × w_i)，total_weight 存放 Σw_i |
| 28-38 | 遍历 4 路传感器 | 每个传感器的权重取决于 mode 参数 |
| 32 | Mode 0 | 直接用 ADC 原始值做权重。注意这里没有对原始值做 normalize 或缩放 |
| 34 | Mode 1 | 超过 threshold 就算 1，否则算 0 |
| 37 | 累加 | `weighted_sum += position × weight` |
| 38 | 累加 | `total_weight += weight` |
| 42-44 | 防除零 | 如果所有权重之和 < 0.001，认为全部没检测到线，返回 0 表示"找不到线，保持当前方向" |
| 47 | 计算归一化偏差 | 加权平均位置 ÷ 3 × 100 |
| 50-51 | 限幅 | 确保输出在 [-100, +100] 范围内，防止 PID 积分项饱或因浮点误差溢出 |

### 为什么 total_weight < 0.001 而不是 < 1？

使用 `0.001f` 作为阈值是因为 Mode 0 下 weight 可以是任意浮点数（ADC 值）。如果所有传感器都读到了很小的值（比如完全在黑线上，反射极低），理论上可能出现 total_weight ≈ 0 但实际有信号的情况。`0.001f` 的设置是保守的。

## 7. 输出特性和 PID 联动

### 7.1 偏差范围

| 条件 | 偏差值 | 含义 |
|------|--------|------|
| 线在所有传感器正中心 | 0 | 完美对中 |
| 线在传感器 3 正上方 | +100 | 最大偏右 |
| 线在传感器 0 正上方 | -100 | 最大偏左 |
| 没检测到线 | 0 | 丢失线 → 需要特殊处理 |

### 7.2 与 PID 的连接

```c
// 典型循迹 loop
float deviation = LineFollower_GetDeviation(sensor_vals, THRESHOLD, 1);

// PID 控制: 偏差为输入，输出目标角速度
float angular_target = pid_calculate(&pid_line, 0.0f, deviation); // 目标=0

// 逆解算轮速
float linear_speed = 0.3f;  // 固定前进速度
DiffDrive_WheelSpeeds(&dd, linear_speed, angular_target, &left, &right);
```

**关键理解**：
- `deviation` 作为 PID 的误差输入（期望值永远为 0，即"对正黑线"）
- PID 输出的 `angular_target` 是目标角速度，正值表示左转、负值右转
- 线速度固定，角速度由 PID 根据偏差大小实时调节

### 7.3 PID 参数调试

```
P 项: 偏差 → 转向强度
    P 太大: 左右剧烈摆动（振荡）
    P 太小: 转弯不足，冲出线外

I 项: 消除稳态偏差（比如线一直偏左 5%）
    I 太大: 响应迟钝，过冲严重

D 项: 抑制振荡，提前预判
    D 太大: 对噪声敏感，抖动
```

推荐起始参数：P=1.0, I=0.0, D=0.1，然后逐步增大 P 直到出现振荡，再回调到 60%，最后微调 D。

## 8. 踩坑点

### 8.1 阈值漂移

**问题**：早上 10 点和下午 4 点的环境光强度不同，同一块白底的 ADC 读数可能差 50%。

**影响**：
- Mode 0：权重值变化，偏差计算受影响
- Mode 1：阈值固定但黑/白底的 ADC 值都变了，可能导致误判

**解决方法**：
- 使用**差分检测**：给灰度传感器加补光灯（红外 LED），使反射光主要来自传感器自身，环境光影响降到最低
- 动态阈值：每次启动时做一次自动标定
- 滑动窗口归一化：取最近 N 个读数的最大最小值做归一化

### 8.2 丢线处理

**问题**：小车急转弯时，线可能完全脱离传感器范围。此时所有传感器读数都接近白底值，total_weight ≈ 0，函数返回 0。

偏差为 0 在 PID 看来是"已经对正了"，不会做转向动作，小车会直行冲出赛道。

**解决方法**：外围代码检测到偏差连续为 0 超过一定时间时：
- 进入"寻线模式"：原地旋转扫描直到找到线
- 或者记录上次偏差方向，继续朝该方向转向

```c
// 丢线检测伪代码
if (fabsf(deviation) < 0.01f && all_white(sensor_vals)) {
    // 确实丢线了, 进入寻线模式
    lost_line_counter++;
    if (lost_line_counter > 5) {
        // 执行寻线动作
    }
} else {
    lost_line_counter = 0;
}
```

### 8.3 传感器串扰

**问题**：相邻传感器之间距离太近，它们的红外发射管会互相干扰。传感器 0 的发射光可能被传感器 1 接收到。

**解决方法**：
- 分时轮流点亮：每次只点亮一个传感器
- 硬件上加遮光隔离

### 8.4 反光地面与黑色反光

**问题**：某些黑色地胶在红外光下反光率很高（类似白色），会导致算法认为黑线是白的，白的反而更暗。

**解决方法**：使用前手动检测一下 sensor 的极性。必要时在代码里取反（`4095 - raw_value`）。

## 9. 调试手段

### 9.1 传感器原始值查看

```c
// 串口打印 4 路传感器值
printf("S0:%4d S1:%4d S2:%4d S3:%4d\r\n",
    sensor[0], sensor[1], sensor[2], sensor[3]);
```

放在黑线上看哪些路数值变小，放在白底上看哪些路数值变大，确认传感器布局正确。

### 9.2 偏差可视化

```c
printf("dev=%.1f\r\n", deviation);
// 在串口绘图器上可以实时看到偏差曲线
// 直线段偏差 ≈ 0, 弯道处偏差产生, PID 响应
```

### 9.3 阈值调试

```c
// 手动标定: 移动小车使每个传感器依次扫过黑白交界
for (int i = 0; i < 4; i++) {
    printf("min[%d]=%d max[%d]=%d\r\n", i, min[i], i, max[i]);
}
// 推荐 threshold = (min_avg + max_avg) / 2
```

### 9.4 PWM 直接听响应

将偏差映射到电机 PWM 增量，听电机声音或者测电压：

```c
// 不通过 PID, 直接 PWM = deviation * K
int left_pwm  = BASE_PWM - (int)(deviation * 5);
int right_pwm = BASE_PWM + (int)(deviation * 5);
```

偏差正（偏右）→ left_pwm 增大, right_pwm 减小 → 左转修正 → 偏差归零。这样可以快速验证偏差计算的方向是否正确。

## 10. 算法局限性

### 10.1 仅适用于单条线

加权重心法假设只有一条线在传感器视野内。如果遇到十字路口、T 字路口，多段黑线会导致加权平均位置在两个黑线之间，产生错误偏差。

### 10.2 无法区分粗细

粗线和细线的重心位置计算都正常，但粗线可能导致多个传感器同时检测到"黑"，使偏差更趋近于 0（重心被拉到中心），削弱了转弯响应。

### 10.3 对传感器间距敏感

传感器间距越大，位置分辨率越差。如果黑线在传感器 1 和 2 之间，Mode 0 可以通过半黑半白的过渡值推断出亚传感器位置；Mode 1 则只能粗糙地判断"不是传感器 1 就是传感器 2"。

## 11. 与其它模块的协作

```
                 LineFollower
                 (偏差计算)
                     |
                     ↓ deviation
              PID 控制器
                     |
                     ↓ (v_target, ω_target)
              DiffDrive
              (逆解轮速)
                     |
                     ↓ (ω_L, ω_R)
              电机 PWM
                     |
                     ↓ 编码器反馈
            dead_reckoning
            (位置估计)
```

LineFollower 是整个控制链的第一个环节——它不做控制决策，只做信号处理。准确度直接决定了循迹的效果上限。
