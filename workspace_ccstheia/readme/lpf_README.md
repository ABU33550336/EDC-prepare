## 1. 这个滤波器是干什么的

一阶低通滤波器(Low-Pass Filter)是**最基础、最常用**的滤波器。它允许低频信号通过,抑制高频噪声。

**在嵌入式中的典型用途**:

- 平滑ADC采样值(去除传感器噪声)
- 平滑PID控制器的误差输入(防止微分项对噪声过度敏感)
- 平滑遥控器/按键输入(去抖)
- 作为互补滤波器/姿态解算中的组成单元

**为什么叫"一阶"?** 因为它的差分方程只依赖上一个输出值(`y[n-1]`),不含更早的项。对应连续域的传递函数分母是s的一次方:

```
        ω_c
H(s) = ─────
       s + ω_c
```

## 2. 结构体定义逐行拆解

```c
typedef struct {
    float alpha;     //滤波系数,0~1,越小越平滑
    float last_out;  //上一次输出值
} LPF_t;
```

**逐行解释**:

| 成员 | 类型 | 含义 | 初始值 |
|------|------|------|--------|
| `alpha` | float | 滤波系数(也称平滑因子) | 用户设定 |
| `last_out` | float | 上一次滤波输出值 | 0.0f |

**LPF_t的设计哲学**: 这是所有滤波器中**最精简**的结构体——只有两个float,8字节。因为在嵌入式系统上,一阶LPF的使用量可能非常大(几十甚至上百个),每个实例少用4字节也很重要。

**关于注释的歧义**: 注释写"alpha越小越平滑"。从公式看:

```c
float out = lpf->alpha * input + (1.0f - lpf->alpha) * lpf->last_out;
```

- alpha小(如0.01) → `1-alpha`大(0.99) → 输出更多地依赖 `last_out`(历史值) → 新输入影响小 → **平滑度极高,响应极慢**
- alpha大(如0.99) → `1-alpha`小(0.01) → 输出更多地依赖 `input`(当前值) → 新输入影响大 → **平滑度低,响应极快**

所以注释无误: **alpha越小越平滑**。

## 3. LPF_Init 逐行拆解

```c
void LPF_Init(LPF_t *lpf, float alpha)
{
    if (lpf == NULL) return;
    if (alpha < 0.0f) alpha = 0.0f;
    if (alpha > 1.0f) alpha = 1.0f;
    lpf->alpha = alpha;
    lpf->last_out = 0.0f;
}
```

**逐行解释**:

- `if (lpf == NULL) return;` — 空指针保护。这是嵌入式C代码的标准防御模式。注意这里静默返回,不设错误码——调用者如果不检查返回值,将永远不知道初始化失败。更好的做法是返回错误码。

- `if (alpha < 0.0f) alpha = 0.0f;` — alpha为负意味着"负反馈",滤波器的输出会反向跟随输入,导致不稳定。钳位到0。

- `if (alpha > 1.0f) alpha = 1.0f;` — alpha超过1意味着当前输入权重 >1,滤波器变成**高通甚至带通**,不再是低通特性。钳位到1。

- `lpf->alpha = alpha;` — 存储alpha。因为alpha范围已经确保在[0,1],这里不会出问题。

- `lpf->last_out = 0.0f;` — **初始输出为0**。这是最重要的初始化陷阱:

  **问题**: 如果实际信号的稳态值不是0(比如传感器偏置在2.5V),从0开始的输出需要若干采样周期才能逐渐逼近真实值。

  **后果**: 前N个滤波输出值偏小,导致启动阶段数据不可用。

  **解决方案**: 如果预先知道信号的初始值,可以让第一个输入直接赋值给last_out:

```c
// 更好的初始化方式
float first_input = ReadSensor();
LPF_Init(&lpf, 0.10f);
lpf.last_out = first_input;  // 绕过启动瞬态
// 或者直接用LPF_Reset
```

## 4. LPF_SetAlpha 逐行拆解

```c
void LPF_SetAlpha(LPF_t *lpf, float alpha)
{
    if (lpf == NULL) return;
    if (alpha < 0.0f) alpha = 0.0f;
    if (alpha > 1.0f) alpha = 1.0f;
    lpf->alpha = alpha;
}
```

**逐行解释**:

与Init中的alpha钳位逻辑完全一致。

**运行时修改alpha的意义**:

- **自适应滤波**: 根据信号特征动态改变平滑程度
  - 信号平稳时 → 减小alpha → 更平滑
  - 信号突变时 → 增大alpha → 更快追踪
- **启动阶段**: 开始时用较大的alpha快速跟踪信号,稳定后切换到小alpha获得平滑输出

## 5. LPF_Reset 逐行拆解

```c
void LPF_Reset(LPF_t *lpf, float value)
{
    if (lpf == NULL) return;
    lpf->last_out = value;
}
```

**逐行解释**:

这个函数只有两个作用:
1. **设置初始值**: `LPF_Reset(&lpf, current_sensor_value)` 可以避开上文说的启动瞬态问题
2. **清除奇异值**: 如果滤波输出出现了不应有的值(如溢出),可以Reset到一个安全的默认值

**注意**: Reset不修改alpha。alpha保留之前的值。意味着你可以在一个地方配好alpha,然后随意Reset。

## 6. LPF_Update 逐行拆解

```c
float LPF_Update(LPF_t *lpf, float input)
{
    if (lpf == NULL) return input;
    float out = lpf->alpha * input + (1.0f - lpf->alpha) * lpf->last_out;
    lpf->last_out = out;
    return out;
}
```

**逐行解释**:

- `if (lpf == NULL) return input;` — 防御性编程。与Init不同,这里**返回原始输入**:即使滤波器无效,也透传数据,保证系统不中断。

- `float out = lpf->alpha * input + (1.0f - lpf->alpha) * lpf->last_out;` — 这是**一阶IIR数字滤波器**的标准递推公式:
  - `lpf->alpha * input` — 当前输入的加权贡献
  - `(1.0f - lpf->alpha) * lpf->last_out` — 上一输出的衰减保留

  两者相加就是滤波后的新输出。

- `lpf->last_out = out;` — 将本次输出保存为`last_out`,供下一次Update使用。这是滤波器**记忆状态**的唯一来源。每次更新都在"忘记"一部分历史、记住一部分新值。

- `return out;` — 返回滤波后的值。

**计算量分析**:
- 2次浮点乘法
- 1次浮点加法
- 1次赋值

总共约5个CPU周期(有FPU时),约0.15μs @ 32MHz。**这大概是你能做到的最快滤波器**。

## 7. 数学公式全解析

### 7.1 离散时间方程

一阶低通滤波器的差分方程为:

```
y[n] = α · x[n] + (1-α) · y[n-1]
```

展开递推关系:

```
y[0] = α · x[0] + (1-α) · y[-1]      (y[-1]是last_out的初始值)
y[1] = α · x[1] + (1-α) · y[0]
y[2] = α · x[2] + (1-α) · y[1]
...
```

代入展开可以得到闭式形式(**加权指数平均**):

```
y[n] = α · x[n] + α·(1-α) · x[n-1] + α·(1-α)² · x[n-2] + ... + (1-α)^{n+1} · y[-1]
```

**关键观察**: 历史输入对当前输出的贡献呈**指数衰减**,衰减因子是 `(1-α)`。这就是为什么它也称为**指数移动平均(Exponential Moving Average, EMA)**。

### 7.2 连续域传递函数

在连续域中,一阶低通滤波器的传递函数是:

```
        ω_c         1
H(s) = ─────  =  ────────
       s + ω_c     1 + τ·s
```

其中:
- ω_c = 2π · f_c 是截止角频率(rad/s)
- τ = 1/ω_c = 1/(2π·f_c) 是时间常数(s)

### 7.3 α与截止频率的换算

一阶数字LPF的截止频率 f_c 与α的关系为(使用双线性变换近似):

```
        α · f_s
f_c ≈ ──────────
        2π · (1-α)
```

其中 f_s = 1/Δt 是采样频率。

**这个公式是一阶LPF调参过程中最重要的公式**。反解出α:

```
         2π · f_c / f_s
α ≈ ──────────────────────
     1 + 2π · f_c / f_s
```

**举例**:

系统采样率 f_s = 100Hz(Δt=0.01s), 希望截止频率 f_c = 5Hz:

```
α ≈ (2π·5/100) / (1 + 2π·5/100)
  ≈ 0.314 / 1.314
  ≈ 0.239
```

| 目标截止频率 | α (f_s=100Hz) | α (f_s=1000Hz) |
|-------------|--------------|---------------|
| 1 Hz | 0.059 | 0.006 |
| 5 Hz | 0.239 | 0.030 |
| 10 Hz | 0.386 | 0.059 |
| 20 Hz | 0.557 | 0.112 |
| 50 Hz | 0.759 | 0.239 |

**显然的结论**: 同样的截止频率,采样率越高则α越小。

## 8. 参数选择指导

### 8.1 确定截止频率的指导原则

**香农采样定理**: 截止频率必须小于采样频率的一半(奈奎斯特频率),否则发生混叠。

但实际应用中,更严格的限制是:

```
f_c ≤ f_s / 10
```

**为什么是1/10而不是1/2?** 一阶滤波器在截止频率处只有-3dB衰减,20dB/dec的滚降。如果要有效抑制50Hz工频干扰,就不能把截止频率设在49Hz——抑制效果只有约3dB。需要让截止频率远低于干扰频率。

**按噪声类型选择**:

| 噪声类型 | 噪声频率 | 推荐f_c | 说明 |
|---------|---------|---------|------|
| ADC白噪声 | 全频带 | f_s/20 ~ f_s/10 | 需要大幅度抑制 |
| 50Hz工频 | 50Hz | 5~10Hz | 远离工频 |
| 机械振动 | 10~100Hz | 2~5Hz | 取决于机械结构 |
| 人体运动 | 0~5Hz | 10~20Hz | 保留运动信号 |
| PID微分噪声 | 高频 | f_s/50 | 尽量平滑 |

### 8.2 α调整步骤

1. **初始化**: 使用保守值 α=0.10
2. **观察**: 看信号是否过度平滑(响应慢)或不够平滑(噪声大)
3. **调整**:
   - 响应慢 → 增大α(增大截止频率)
   - 噪声大 → 减小α(降低截止频率)
4. **重复**: 直到找到不同应用场景之间的平衡点

## 9. 典型调用链

```
// 场景1: 平滑ADC采样值
ADC_Read()
  └─> LPF_Update(&adc_lpf, raw_value)
       └─> 返回平滑后的值给应用层

// 场景2: PID误差输入滤波
PID_Controller()
  ├─> error = setpoint - feedback
  ├─> filtered_error = LPF_Update(&pid_lpf, error)  // 误差先滤波再进PID
  └─> output = Kp * filtered_error + Ki * integral + Kd * derivative

// 场景3: 遥控器信号去抖
RemoteControl_Decode()
  ├─> raw_channel = ReadRC()
  ├─> smooth_channel = LPF_Update(&rc_lpf, raw_channel)
  └─> 用smooth_channel控制舵机/电机
```

**被谁调用**:

| 调用位置 | 滤波对象 | 推荐α |
|---------|---------|-------|
| PID误差输入 | 控制误差值 | 0.10 ~ 0.30 |
| ADC采集后处理 | 传感器原始值 | 0.05 ~ 0.20 |
| 遥控器信号 | PPM/PWM占空比 | 0.30 ~ 0.50 |
| 姿态角输出 | 角度(二级滤波) | 0.50 ~ 0.80 |
| 电机电流采样 | 电流值 | 0.01 ~ 0.10 |

## 10. 踩坑点大全

### 10.1 坑1: α太接近1导致输出延迟大

**现象**: α=0.999,输出几乎不跟随输入变化,像一个"反应迟钝"的系统。

**分析**: α=0.999意味着(1-α)=0.001。截止频率:

```
f_c = α·f_s / (2π·(1-α)) = 0.999·100 / (2π·0.001) ≈ 15.9kHz @100Hz采样
```

截止频率已经远高于采样率,实际上几乎没有滤波效果——但同时输出几乎不跟随输入,因为99.9%的权重在历史值上。

**真正的问题**: α越接近1,历史值的衰减越慢,启动瞬态越长。α=0.999时,时间常数:

```
τ = Δt / (1-α) = 0.01 / 0.001 = 10秒
```

需要 **3τ = 30秒** 输出才稳定。这在调试时非常容易让人困惑:"为什么我的滤波器参数设对了但输出总是不对?"

**建议**: α不要超过0.99,除非你很清楚自己在做什么。

### 10.2 坑2: float精度导致的饱和效应

**现象**: 滤波器长时间运行后,输出与输入不再准确对应。

**分析**: float有24位尾数(约7位十进制有效数字)。当 `last_out` 已经积累到很大值(如10000),此时输入一个很小的变化(如0.01):

```
out = 0.1 * 0.01 + 0.9 * 10000.0
    = 0.001 + 9000.0
    = 9000.001
```

float能表示的最小增量是 `|last_out| × 2^-24`, 对于9000来说约0.0005,所以0.001的变化可以正确累加。但如果输入变化更小(<0.0005),就会丢失。

**解决方案**: 如果信号范围很大(如电机编码器累计脉冲),应该用定点数或double。

### 10.3 坑3: 输入突变导致输出暂态

**现象**: 输入从0突然跳到100,输出以指数形式慢慢爬升。

**分析**: 这不是bug,这是一阶滤波器的**固有特性**: 任何突变都会被"抹平"。但对于某些应用(如急停信号),这种延迟不可接受。

**解决方案**:

```c
// 带门限的LPF: 突变超过阈值时直通
float LPF_Update_WithThreshold(LPF_t *lpf, float input, float threshold)
{
    if (fabsf(input - lpf->last_out) > threshold) {
        lpf->last_out = input;  // 突变直通,跳过滤波
        return input;
    }
    return LPF_Update(lpf, input);
}
```

### 10.4 坑4: 多级串联导致相位滞后累加

**现象**: 两个LPF串联(输出再接一个LPF),总延迟变成单级的平方。

**分析**: 两级LPF串联,相位响应叠加,-3dB带宽降低约36%,延迟增加约一倍。如果系统中多处使用LPF(如传感器滤波→PID误差滤波→输出滤波),总的延迟可能超过100ms。

**解决方案**: 尽量减少串联级数,或者提高每一级的截止频率来补偿。

### 10.5 坑5: 调用频率不固定

**现象**: LPF_Update在中断中调用时频率固定,但在主循环中调用时频率随CPU负载变化。

**影响**: 同样的α在不同调用频率下,实际截止频率不同。

**解决方案**: 要么确保固定调用频率,要么使用与时间相关的LPF版本:

```c
// 与时间相关的LPF,dt由外部传入
float LPF_Update_DT(LPF_t *lpf, float input, float dt, float tau)
{
    float alpha = dt / (tau + dt);  // 根据实际时间计算alpha
    if (alpha > 1.0f) alpha = 1.0f;
    float out = alpha * input + (1.0f - alpha) * lpf->last_out;
    lpf->last_out = out;
    return out;
}
```

## 11. 初始化与启动瞬态

### 11.1 启动暂态定量分析

假设输入为恒定值X,初始输出y[-1]=0。第n次输出的表达式:

```
y[n] = X · [1 - (1-α)^{n+1}]
```

稳定时间以`误差降到1%以内`为准:

```
(1-α)^{n+1} ≤ 0.01
n ≥ ln(0.01) / ln(1-α) - 1
```

| α | (1-α) | 99%稳定所需次数 | 对应时间(f_s=100Hz) |
|---|-------|----------------|-------------------|
| 0.01 | 0.99 | 458次 | 4.58s |
| 0.05 | 0.95 | 90次 | 0.90s |
| 0.10 | 0.90 | 44次 | 0.44s |
| 0.20 | 0.80 | 21次 | 0.21s |
| 0.50 | 0.50 | 7次 | 0.07s |

**关键结论**: α=0.01时,你需要等待近5秒才能得到有效输出。如果你的系统在启动后需要立即响应,绝对不能用小α。

### 11.2 绕过启动瞬态的方法

```c
// 方法1: 用第一次输入值初始化
float first_val = ReadSensor();
LPF_Init(&lpf, 0.05f);
lpf.last_out = first_val;  // 直接跳过启动瞬态

// 方法2: 使用重置函数
LPF_Reset(&lpf, ReadSensor());

// 方法3: 动态alpha(启动时大,稳定后小)
LPF_Init(&lpf, 0.50f);  // 快速跟踪
for (int i = 0; i < 20; i++) {
    LPF_Update(&lpf, ReadSensor());
}
LPF_SetAlpha(&lpf, 0.05f);  // 切到平滑模式
```

## 12. 调试手段

### 12.1 串口打印对比

同时打印原始数据和滤波后数据,用串口绘制双线对比:

```c
float raw = ReadSensor();
float filtered = LPF_Update(&lpf, raw);
printf("%lu,%f,%f\n", micros(), raw, filtered);
```

**观察要点**:
- 原始曲线的噪声幅度
- 滤波后曲线是否有效抑制了噪声
- 滤波后曲线是否出现过大的延迟(相位滞后)

### 12.2 步进响应测试

输入一个阶跃信号(从A突然变到B),观察滤波输出的响应曲线:

理论上,输出应该是指数上升/下降:

```
y(t) = B + (A - B) · e^{-t/τ}
```

实际调试时:

1. 给定阶跃输入(如从0→100)
2. 串口记录原始值和滤波值
3. 绘制曲线,测量从10%到90%的上升时间 t_r
4. 计算时间常数 τ ≈ t_r / 2.2
5. 反推截止频率 f_c = 1/(2π·τ)

如果t_r比你期望的大(响应慢),增大α;如果噪声抑制不够,减小α。

### 12.3 动态调参

通过串口命令在运行时修改α,实时观察效果:


## 13. 为什么都用float

与complementary_filter的说明一致:

1. **MSPM0G3507有单精度FPU**: float运算等价于整数运算速度
2. **精度足够**: 传感器ADC通常是12~16位,float的24位尾数绰绰有余
3. **省RAM**: 每个实例只占8字节(2×float)
4. **代码简洁**: 不需要定点数转换的繁琐操作

如果用double:
- 每个实例占16字节
- 没有硬件加速,靠软件模拟,慢数十倍
- 多出的精度毫无意义(传感器噪声本身远大于float的量化误差)

如果用定点数:
- 需要手动处理小数位的对齐(Q格式)
- 需要特别注意溢出的问题
- 代码可读性差,难以维护

**结论**: float是嵌入式传感器滤波的"甜点"选择。
