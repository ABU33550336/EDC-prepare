## 1. 这个滤波器是干什么的

中值滤波器(Median Filter)是一种**非线性**滤波器,原理极其简单: 对一个滑动窗口内的数据排序,取**中间的那个值**作为输出。

**为什么中值滤波有效**? 因为噪声(特别是脉冲噪声/椒盐噪声)往往出现在序列的极端值位置,排序后被挤到窗口的两端,中间值不受影响。

**在嵌入式中的典型用途**:

| 应用 | 说明 |
|------|------|
| 传感器去毛刺 | 红外测距传感器受反射干扰产生的异常跳变 |
| 按键去抖 | 机械按键按下瞬间的弹跳噪声 |
| 图像处理 | 去除图像中的椒盐噪声(但本模块是1D的) |
| ADC采样 | 去除ADC偶发的异常采样值(电源毛刺) |

## 2. 中值滤波 vs 均值滤波

这是一个**永恒的比较**。两者都是滑动窗口滤波器,但原理天差地别:

| 维度 | 中值滤波 | 均值滤波 |
|------|---------|---------|
| **类型** | 非线性 | 线性 |
| **对脉冲噪声** | ✅ 完全消除 | ❌ 脉冲扩散到整个窗口 |
| **对高斯噪声** | ❌ 效果一般(统计效率~64%) | ✅ 最优(最小方差估计) |
| **对阶跃信号** | ✅ 保持边缘锐度 | ❌ 阶跃被抹平为斜坡 |
| **计算量** | O(N²)或O(N·logN) | O(1)(用滑动和) |
| **RAM需求** | 2×N×float | N×float(或用滑动和只需2个) |
| **输出延迟** | (N-1)/2 | (N-1)/2 |

**什么时候用中值**:
- 信号中偶尔有**大幅度毛刺**(超声波、红外传感器)
- 需要保持信号的**边沿/跳变**(编码器计数、PWM占空比检测)

**什么时候用均值**:
- 噪声是**高斯分布**的(热噪声、ADC量化噪声)
- 对实时性要求高(均值可以用滑动窗口O(1)实现)

## 3. 结构体定义逐行拆解

```c
typedef struct {
    float *buffer;    //数据缓冲区
    float *sort_buf;  //排序缓冲区,用于取中值
    uint16_t size;    //窗口大小
    uint16_t index;   //当前写入位置
    uint16_t count;   //已采样数量
} MedianFilter_t;
```

**逐行解释**:

| 成员 | 类型 | 含义 | 为什么需要 |
|------|------|------|-----------|
| `buffer` | float* | 环形数据区,存原始采样序列 | 维护窗口内所有数据 |
| `sort_buf` | float* | 排序用的临时缓冲区,与buffer等长 | 排序时复制一份,不破坏buffer的原始顺序 |
| `size` | uint16_t | 窗口大小(元素个数) | 必须≥3才能有中值 |
| `index` | uint16_t | 当前环形缓冲区写入位置 | 实现FIFO的环状覆盖 |
| `count` | uint16_t | 已经采样的数量(启动阶段不到size) | 窗口未填满时用均值替代 |

**显然的问题**: 为什么buffer和sort_buf都是**指针**而不是数组?

因为中值滤波器每次Update都要对 **整个窗口排序**,如果窗口较大(如size=10),每次在栈上分配数组既慢又浪费。用指针让调用者提供静态或全局缓冲区:

```c
// 调用者提供的静态缓冲区
static float mf_buf[10];
static float mf_sort[10];
MedianFilter_t mf;
MedianFilter_Init(&mf, mf_buf, mf_sort, 10);
```

这样:
- 缓冲区内存由调用者管理
- 同一个滤波器实例可以反复使用
- 不会出现栈溢出(特别是中断中用滤波器时)

## 4. insertion_sort 逐行拆解

```c
static void insertion_sort(float *arr, uint16_t len)
{
    for (uint16_t i = 1; i < len; i++) {
        float key = arr[i];
        int16_t j = (int16_t)i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

### 4.1 为什么用插入排序?

中值滤波器的窗口大小通常是**3、5、7、9**等小奇数。对于小规模数据排序,插入排序是最合适的:

| 排序算法 | 时间复杂度 | 小数组表现 | 是否稳定 | 就地排序 |
|---------|-----------|-----------|---------|---------|
| 插入排序 | O(N²) | 极好(常数极小) | ✅ 稳定 | ✅ |
| 冒泡排序 | O(N²) | 差(常数大) | ✅ | ✅ |
| 快速排序 | O(N·logN) | 差(递归开销大) | ❌ | ✅ |
| 选择排序 | O(N²) | 中 | ❌ | ✅ |

对于N=3~9的数组,插入排序的实际指令数最少。插入排序在**几乎有序**的数组上甚至可以达到O(N)——而中值滤波的窗口数据恰好有"时间连续性",相邻两次采样的窗口数据只差一个元素,所以**sort_buf实际上是有序或接近有序的**。

### 4.2 逐行解释

```c
for (uint16_t i = 1; i < len; i++) {
```

插入排序从**第2个元素**(索引1)开始,依次把每个元素插入到前面已排序序列的正确位置。

```c
    float key = arr[i];
```

取出当前要插入的元素,称为"键"(key)。保存到临时变量中,因为它在数组中的位置可能会被其他元素覆盖。

```c
    int16_t j = (int16_t)i - 1;
```

j指向已排序区域的最后一个元素。注意这里用的`int16_t`,因为j需要变成-1来表示已到达数组头部。

**类型转换的微妙性**: `i`是`uint16_t`,如果不强转为`int16_t`,当i=0时`i-1`会下溢为65535。所以必须用 `(int16_t)i - 1`。

```c
    while (j >= 0 && arr[j] > key) {
        arr[j + 1] = arr[j];
        j--;
    }
```

**循环条件**:
- `j >= 0` — 确保未超出数组左边界
- `arr[j] > key` — 如果已排序元素大于key,就把该元素右移一位

**右移**: `arr[j + 1] = arr[j]` 覆盖了key原本的位置(或者是之前被右移空出来的位置)。

```c
    arr[j + 1] = key;
```

循环结束时,j指向最后一个≤key的元素(或-1),在这个位置后面插入key。

### 4.3 示例: 用插入排序对 [3, 1, 4, 1, 5] 排序

```
初始: [3, 1, 4, 1, 5]
i=1: key=1, j=0, arr[0]=3>1 → arr[1]=3, j=-1 → arr[0]=1 → [1, 3, 4, 1, 5]
i=2: key=4, j=1, arr[1]=3<4 → 不进入while → arr[2]=4 → [1, 3, 4, 1, 5]
i=3: key=1, j=2, arr[2]=4>1 → arr[3]=4, j=1; arr[1]=3>1 → arr[2]=3, j=0; arr[0]=1≤1 → arr[1]=1 → [1, 1, 3, 4, 5]
i=4: key=5, → [1, 1, 3, 4, 5]
```

## 5. MedianFilter_Init 逐行拆解

```c
uint8_t MedianFilter_Init(MedianFilter_t *mf, float *buf,
                          float *sort_buf, uint16_t size)
{
    if (mf == NULL || buf == NULL || sort_buf == NULL) return 1;
    if (size < 3) return 1;
    mf->buffer   = buf;
    mf->sort_buf = sort_buf;
    mf->size     = size;
    mf->index    = 0;
    mf->count    = 0;
    for (uint16_t i = 0; i < size; i++) {
        buf[i] = 0.0f;
    }
    return 0;
}
```

**逐行解释**:

- `if (mf == NULL || buf == NULL || sort_buf == NULL) return 1;` — **三个指针都必须有效**。与之前的一阶滤波器不同,这个函数的返回值是有意义的:0=成功,1=失败。

- `if (size < 3) return 1;` — 窗口最小为3。size=1时中值就是自身,滤波无意义;size=2时没有中间的"那个"值(取平均值也行但违背了中值滤波的初衷)。

- `mf->buffer = buf;` `mf->sort_buf = sort_buf;` — 保存外部缓冲区指针。不复制数据,只是存引用。

- `mf->size = size;` — 窗口大小,后续Update依赖此值进行环形写回绕和排序。

- `mf->index = 0;` — 写入位置从0开始。

- `mf->count = 0;` — 采样计数从0开始。这是"窗口未填满"标志。

- `for (uint16_t i = 0; i < size; i++) { buf[i] = 0.0f; }` — 清空缓冲区。如果不清零,缓冲区中的初始值是随机的,前几次中值输出会乱跳。

- `return 0;` — 初始化成功。

## 6. MedianFilter_Reset 逐行拆解

```c
void MedianFilter_Reset(MedianFilter_t *mf)
{
    if (mf == NULL || mf->buffer == NULL) return;
    mf->index = 0;
    mf->count = 0;
    for (uint16_t i = 0; i < mf->size; i++) {
        mf->buffer[i] = 0.0f;
    }
}
```

**逐行解释**:

与Init逻辑几乎一致,但:
- 不重置`size`和`sort_buf`指针——这些在Init时一次性设定,Reset不修改配置
- 清空buffer,归零index和count

**注意**: Reset不归零`sort_buf`,因为sort_buf只在Update中被使用,它的内容是临时的。

## 7. MedianFilter_Update 逐行拆解

```c
float MedianFilter_Update(MedianFilter_t *mf, float input)
{
    if (mf == NULL || mf->buffer == NULL || mf->size < 3) {
        return input;
    }
```

**入口检查**: 任何条件不满足就透传输入值。注意这里没有检查`sort_buf`——虽然Init时要求它非空,但Update没有验证。这是一个小缺陷。

```c
    mf->buffer[mf->index] = input;           //写入环形缓冲区
    mf->index++;
    if (mf->index >= mf->size) {             //到达末尾则回绕
        mf->index = 0;
    }
```

**环形缓冲区写入**: 每次写入当前位置后index自增,当index到达size时回绕到0。这是最基本的**环形队列FIFO**实现。

**为什么用环形缓冲区?** 因为中值滤波器的窗口是"滑动的": 新数据来的时候,最旧的数据被丢弃。用环形缓冲区:
- 不需要搬移数据(O(1))
- 空间固定(size个float)
- 读写都在O(1)完成

```c
    if (mf->count < mf->size) {
        mf->count++;
    }
```

**采样计数**: 从0开始,最大到size。当count=size时,窗口已满,此后一直保持满状态。

```c
    if (mf->count < mf->size) {              //窗口未填满时求平均,避免中值偏差
        float sum = 0.0f;
        for (uint16_t i = 0; i < mf->count; i++) {
            sum += mf->buffer[i];
        }
        return sum / (float)mf->count;
    }
```

**窗口未满时的处理**:

这是个非常重要的设计决策。为什么不直接取中值?

**原因**: 当窗口未填满时,缓冲区中有很多0.0f(初始值)。如果对这些0排序取中值,输出会被拉到0,产生严重偏差。

例如size=5,只有前3个数据是[10, 12, 11],后两个是[0, 0]。排序得到[0, 0, 10, 11, 12],中值是10——这显然不对,真实数据的趋势被两个0破坏了。

**所以**: 窗口未满时,用**均值**代替中值。等窗口填满后再用中值。

```c
    memcpy(mf->sort_buf, mf->buffer, mf->size * sizeof(float));
    insertion_sort(mf->sort_buf, mf->size);
    return mf->sort_buf[mf->size / 2];       //取排序后中间值作为滤波输出
}
```

**窗口满时的中值计算**:

- `memcpy(...)` — 把buffer**完整复制**到sort_buf。这是保护原始顺序的关键——因为后续的插入排序会**就地修改数组元素**。

- `insertion_sort(mf->sort_buf, mf->size);` — 对sort_buf排序。

- `mf->sort_buf[mf->size / 2]` — **取中值**。对于奇数size(3,5,7...):
  - size=3: index = 1(第2个)
  - size=5: index = 2(第3个)
  - size=7: index = 3(第4个)

## 8. 数学原理全解析

### 8.1 中值的定义

对于有序序列 x₀ ≤ x₁ ≤ ... ≤ x_{N-1}:

```
            ┌ x_{(N-1)/2}           如果N为奇数
Median(x) = ┤
            └ (x_{N/2-1} + x_{N/2})/2  如果N为偶数
```

本模块只支持奇数窗口(最小3),所以始终取中间一个元素。

### 8.2 中值滤波的统计特性

**概率密度角度**: 中值是分布函数的50%分位点。对于**拉普拉斯分布**(双指数分布,脉冲噪声的常见模型),中值是**最小方差无偏估计**。

**对比**: 均值是最小方差估计——但仅当噪声是高斯分布时才最优。如果噪声是脉冲状的(具有重尾分布),均值会被极端值拉偏,而中值不受影响。

**效率**: 在高斯噪声下,中值的方差约是均值的1.57倍(效率64%)。也就是说,为了达到相同的滤波效果,中值滤波需要约1.57倍的窗口大小。

### 8.3 中值滤波的频率响应

**中值滤波是非线性的,没有传递函数**。它的"频率响应"概念是用**阶跃响应**和**根均方误差(RMSE)**来表征的:

- **对阶跃信号**: 中值滤波可以**无衰减地保留阶跃**(只要阶跃两侧的数据量小于窗口的一半)
- **对正弦信号**: 会产生非线性失真(削波现象)
- **对脉冲**: 当脉冲宽度 < (N-1)/2 时,脉冲被完全滤除

## 9. 参数选择指导

### 9.1 窗口大小选择

中值滤波器**只有一个参数**: 窗口大小N(必须是奇数)。

**选择原则**: 窗口越大→越平滑→延迟越大→计算量越大

| 窗口N | 延迟(采样数) | 最大消除脉冲宽度 | 计算量 | 适用场景 |
|-------|------------|----------------|-------|---------|
| 3 | 1 | 1个采样 | 极小 | 轻度去毛刺 |
| 5 | 2 | 2个采样 | 小 | 一般传感器去噪 |
| 7 | 3 | 3个采样 | 中 | 较强噪声抑制 |
| 9 | 4 | 4个采样 | 中 | 噪声严重的传感器 |
| 15+ | 7+ | 7+个采样 | 大 | 极少用,延迟太大 |

**显然的规律**: 延迟 = (N-1)/2, 可消除的孤立脉冲宽度 = (N-1)/2。

**举例**: 如果信号中有一个宽度为3个采样的脉冲,窗口N=3时无法消除(脉冲占满了窗口),N=5时可以消除(脉冲只占3/5)。

### 9.2 多级中值滤波

如果需要更强的滤波效果但不希望增加延迟,可以用**多级级联**:

```c
// 串联两个size=3的中值滤波器,等效延迟2而不是(size-1)/2=2
// 但性能与size=5接近
float v1 = MedianFilter_Update(&mf1, input);
float v2 = MedianFilter_Update(&mf2, v1);
```

### 9.3 混合滤波

一种常见的做法是: **中值滤波 + 均值滤波**结合:

```c
// 先中值去毛刺,再均值去高斯噪声
float median_out = MedianFilter_Update(&mf, raw);
float final_out = LPF_Update(&lpf, median_out);
```

这比单纯增大窗口更有效,因为两种滤波器针对不同噪声类型。

## 10. 典型调用链

```
// 场景: 超声波测距去毛刺
while(1)
{
    uint16_t distance = ReadUltrasonic();  // 读超声波,可能因声波反射产生异常值

    // 中值滤波去毛刺

    // 如果距离突变超过阈值才认为有效
```

| 调用位置 | 处理的数据 | 推荐窗口 |

| ADC采样(电源噪声环境) | 传感器模拟值 | 5 |
| 霍尔传感器脉冲计数 | RPM值 | 3 |

## 11. 踩坑点大全

### 11.1 坑1: 排序破坏原始时序

**现象**: 使用中值滤波后输出值**出现时间上的错乱**,在某些突发信号场景下表现诡异。

**根因**: 中值滤波输出的是**窗口内的中间值**,不一定是当前采样点对应的值。例如窗口大小为5,最新的采样值是100,但中值输出可能是窗口内第3个采样(比如80)。

**后果**: 输出信号相对于输入信号有**随机的时序偏移**,不是固定的延迟,而是取决于信号在窗口内的分布。

**解决方案**: 理解中值滤波的"延迟"是不确定的(N-1)/2,它反映的是窗口中心的值,而不是最新值。

### 11.2 坑2: 窗口边界处理不当

**现象**: 滤波器刚开始工作时,输出值剧烈跳动。

**根因**: 窗口未满时的均值替代方案是合理的,但如果初始值0与实际信号范围差异很大(如信号范围-30~-10,0在这个范围内不算异常),"均值替代中值"的策略反而引入了偏差。

**更好的做法**: 在Init时用第一个输入值填充整个缓冲区:

```c
void MedianFilter_Init_Fill(MedianFilter_t *mf, float *buf, float *sort_buf, uint16_t size, float init_val)
{
    MedianFilter_Init(mf, buf, sort_buf, size);
    for (uint16_t i = 0; i < size; i++) {
        buf[i] = init_val;
    }
    mf->count = size;  // 直接标记为满,跳过启动阶段
}
```

### 11.3 坑3: 输入全是相同值时排序没问题

**现象**: 当输入恒定时,每次排序都产生完全一样的输出,但计算量不减。

**分析**: 插入排序在已排序(或几乎有序)序列上效率很高,但恒定值导致每次仍然要执行N次比较。这个计算开销对于小窗口可以忽略。

### 11.4 坑4: 浮点相等比较的问题

排序中用的是 `arr[j] > key` (严格大于),不处理相等情况。相等时不会交换,保持了**稳定性**(先出现的元素排前面)。这对于float来说通常没问题,因为精确相等的浮点数不多见。

### 11.5 坑5: 窗口大小的奇偶选择

**这个模块只支持奇数窗口**。如果你的应用需要偶数窗口,中值取中间两个值的平均,但本模块没有实现这个功能。

### 11.6 坑6: 实时性对排序时间的约束

**问题**: 中值滤波的计算时间不稳定,每次Update都要排序。窗口越大,排序耗时越长。

```
N=3:  最坏3次比较,0次交换
N=5:  最坏10次比较,4次交换
N=7:  最坏21次比较,6次交换
N=9:  最坏36次比较,8次交换
```

在MSPM0G3507 @ 32MHz:
- N=3: ~1μs
- N=5: ~3μs
- N=7: ~6μs
- N=9: ~10μs

这些时间在绝大多数应用中可以忽略。

## 12. 初始化与启动瞬态

### 12.1 启动阶段的输出变化

启动时count从0到size的变化过程:

| 第几次Update | count | 缓冲区状态 | 输出方式 | 输出值 |
|-------------|-------|-----------|---------|-------|
| 1 | 1 | [x0,0,0,0,0] | 均值 | x0/1 = x0 |
| 2 | 2 | [x0,x1,0,0,0] | 均值 | (x0+x1)/2 |
| 3 | 3 | [x0,x1,x2,0,0] | 均值 | (x0+x1+x2)/3 |
| 4 | 4 | [x0,x1,x2,x3,0] | 均值 | (x0+x1+x2+x3)/4 |
| 5 | 5 | [x0,x1,x2,x3,x4] | 中值 | sort的中值 |

**所以**: 前(size-1)次输出都是均值,第size次开始输出中值。

### 12.2 启动瞬态时间

如果信号是稳定值C:
- 前4次输出: C, C, C, C(均值)
- 第5次输出: C(中值)
- 实际从第1次开始就是正确的

如果信号是变化的:
- 前4次输出是前N个值的平均,可能与最新值有偏差
- 第5次后输出窗口的中值

## 13. 调试手段

### 13.1 串口打印对比

```c
float raw = ReadSensor();
float median = MedianFilter_Update(&mf, raw);

// 同时打印原始值、中值、缓冲区原始数据(调试用)
printf("%lu,%f,%f,", micros(), raw, median);
for (int i = 0; i < mf.size; i++) {
    printf("%f,", mf.buffer[i]);
}
printf("\n");
```

**观察要点**:
- 原始值的毛刺是否在滤波后消失
- 中值输出是否"滞后"于输入(正常约为(N-1)/2个采样)
- 单个毛刺是否导致中值跳动(如果是,说明窗口不够大)

### 13.2 模拟脉冲测试

在调试阶段,可以人工注入脉冲:

```c
// 正常值50,在第5次时突变为200
for (int i = 0; i < 20; i++) {
    float test_val = (i == 5) ? 200.0f : 50.0f;
    float out = MedianFilter_Update(&mf_test, test_val);
    printf("%d,%f,%f\n", i, test_val, out);
}
```

期望结果(N=5):
```
i=0: 200被注入,输出均值 200/1=200  (启动阶段,count=1)
i=1: 50, 输出 (200+50)/2=125
i=2: 50, 输出 (200+50+50)/3=100
i=3: 50, 输出 (200+50+50+50)/4=87.5
i=4: 50, 窗口满,中值 = sort[50,50,50,50,200] = 50  ← 毛刺完全消除
i=5: 50, 窗口滑动,旧200被覆盖,中值=50  ← 之后一直稳定在50
```

注意: 第0次输出的200不是毛刺,是"初始瞬态"的一部分。真正验证毛刺消除,要看i=4之后的表现。

## 15. 排序算法为什么用插入排序

### 15.1 小数组的王者

插入排序的**实际指令数**:

```
比较次数: N(N-1)/2
交换次数: N(N-1)/2
```

对于N=5: 最多10次比较,10次赋值。对比快速排序: 递归调用、栈操作、基准值选择……在小数组上插入排序更快。

### 15.2 利用局部有序性

中值滤波的相邻两次Update,窗口数据只变了**1个元素**(新数据替换最旧数据)。复制到sort_buf后,sort_buf接近有序,插入排序在几乎有序的数组上运行时间接近O(N)。

### 15.3 为什么不直接用`qsort`

标准库的`qsort`:
- 函数指针调用开销大(每次比较都通过函数指针间接调用)
- 递归栈开销
- 对于小数组,初始化开销远大于排序本身

**结论**: 对于窗口大小3~9,插入排序是最高效的选择。

## 16. 为什么用两个缓冲区

这是初学者最容易忽略的设计决策。

**buffer**: 维护**原始采样顺序**,每次新采样写入buffer[index],覆盖最旧的数据。buffer中的数据是**按时间顺序排列**的。

**sort_buf**: 排序时复制buffer的数据,排序后取中值。排序**只修改sort_buf**,不碰buffer。

**如果只用一个缓冲区**:

当对buffer就地排序后,buffer中的原始时序被破坏。下一次Update时,无法知道"哪个是最旧的数据"——因为数据已经重新排列了,index指向的不再是最旧值。

**因此**: 两个缓冲区是必要的。一个存时序,一个存排序。

**缺点**: RAM占用翻倍。对于嵌入式系统来说,两个窗口数据就是2×N×4字节。N=5时只有40字节,影响很小。但如果窗口很大(如N=100,图像处理),就应该用其他方法(如直方图法)。
